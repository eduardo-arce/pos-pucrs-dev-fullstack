# 🚨Arquitetura-Server-Side🚨

# Sobre a Disciplina

Estudo da arquitetura client-server voltada a aplicações web, com atenção a frameworks MVC server-side, programação assíncrona e reativa, desenvolvimento orientado a serviços e boas práticas como testes unitários e validação de dados.

# Conceitos chave

- Arquitetura client-server: modelo no qual front-end e back-end se comunicam para entregar a aplicação web  
- Frameworks MVC server-side: Node.js, Express e NestJS organizando o código em Model, View e Controller  
- Programação assíncrona: forma de conduzir tarefas simultâneas sem travar o fluxo principal  
- Programação reativa: baseada em fluxos assíncronos de dados, com fundamentos como elasticidade, message drive, responsividade e resiliência  
- Microsserviços versus monolitos: serviços independentes e escaláveis com APIs definidas ou tudo acoplado em um único aplicativo  
- Testes unitários: validação de funções isoladas, garantindo feedback rápido e prevenindo bugs  
- Validação: uso de bibliotecas como Yup para esquemas e checagem de dados  

# Ferramentas iniciais

Node.js, Express, NestJS, Yup (validação) e ferramentas de testes unitários como Jest
