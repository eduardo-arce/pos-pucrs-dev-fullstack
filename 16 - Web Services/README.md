# 🚨Serviços-Web-Web-Services🚨

# Sobre a Disciplina

Disciplina voltada aos web services, abordando arquitetura monolítica, revisão de SOAP, REST, GraphQL e descritores de serviço, além de soluções serverless e construção de soluções com frameworks REST e GraphQL.

# Conceitos chave

- Web services: serviços independentes para comunicação entre sistemas, evoluindo da internet (ARPAnet) até a nuvem  
- Arquitetura monolítica: aplicação única onde todas as funcionalidades estão acopladas  
- SOAP: protocolo baseado em XML para troca de mensagens estruturadas  
- REST: estilo arquitetural sobre HTTP, com recursos representados por URIs e stateless  
- GraphQL: linguagem de consulta para APIs que possibilita queries precisas e evita over/under-fetching do REST  
- Serverless: execução de código sem administrar servidores, com foco em funções (FaaS)  
- Twelve Factor App: princípios para aplicações escaláveis, incluindo base única de código, dependências explícitas, configuração via ambiente, backing services e separação entre build, release e run  

# Ferramentas iniciais

Frameworks REST (Node.js/Express), GraphQL (Apollo), descritores (OpenAPI e WSDL), recursos da W3C e testes unitários
