# Fundamentos da Computação e Algoritmos

## Sobre a disciplina

Primeira imersão nos pilares da computação dentro da pós, esta cadeira combina o raciocínio lógico com a prática hands-on em algoritmos e estruturas de dados. O objetivo é desenvolver uma maneira estruturada de resolver problemas, compreender o custo das decisões tomadas e escolher a ferramenta certa para cada cenário — da simples iteração em um array até a recursão sobre uma árvore binária.

A linguagem escolhida como veículo de aprendizado é o **JavaScript**, justamente pela baixa barreira de entrada (roda direto no navegador) e pela ponte natural com as disciplinas de front-end que vêm depois.

## Temas centrais trabalhados

A disciplina percorre, entre outros tópicos:

- Lógica de programação e estruturação de soluções.
- Estruturas de controle (condicionais) e laços de repetição.
- Tipos de dados primitivos e variáveis.
- Funções e procedimentos.
- Análise de complexidade (Big-O) e comparação entre melhor, médio e pior caso.
- Estruturas de dados: Array, Pilha (LIFO), Fila (FIFO), Lista Encadeada, Map, Set e Árvore Binária.
- Técnicas de recursão.
- Algoritmos de busca (linear e binária) e de ordenação.
- Discussões sobre performance real, latência e _perceived performance_.
- Rotina prática: programação no navegador, VS Code, Git/GitHub, depuração e refactoring.

## Exercícios

As atividades práticas estão divididas entre os módulos conduzidos por cada professor:

### Módulo Mário Souto

- **calcula-idade** — primeiro contato com entrada/saída e raciocínio algorítmico em JavaScript.
- **app-corrida** — exercício um pouco mais robusto: entrada, processamento e saída aplicados a um cálculo de corrida.
- **mini-browser** — simulação de um navegador minimalista para exercitar, na prática, os conceitos de pilha (histórico) e fila.

### Módulo Edson Moreno

- **algoritmo-paridade** — verificação se um número é par ou ímpar.
- **algoritmo-set** — explorando a estrutura `Set` nativa do JavaScript.
- **algoritmo-lista-encadeada** — implementação de uma linked list do zero.
- **algoritmo-recursao** — cálculo de fatorial via recursão, um clássico para internalizar a ideia de chamada recursiva.

## Ferramental apresentado

Durante as aulas surgem algumas ferramentas e referências que vale destacar:

- **Quokka.js** — plugin do VS Code que permite _live coding_ e prototipação rápida em JavaScript/TypeScript, exibindo resultados inline enquanto o código é escrito.
- **Datomic** — banco de dados cronológico (imutável e versionado), trazido como curiosidade arquitetural.
- **GTmetrix** — medidor de desempenho de páginas web, útil para conectar a discussão de complexidade algorítmica com impacto real no usuário.
- **VS Code** — IDE adotada como padrão no curso.
