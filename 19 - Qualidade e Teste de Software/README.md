# 🚨Qualidade-e-Testes-de-Software🚨

# Sobre a Disciplina

A disciplina apresenta os conceitos de testes unitários, de integração e de UI, reforçando garantia de qualidade de software e o uso de métricas para controle de qualidade durante todo o processo de desenvolvimento.

# Conceitos chave

- Testes unitários: verificam funções isoladas, com stubs e esqueletos para garantir isolamento, além de estratégias como TDD (codificar-testar-projetar) e BDD (foco em componente isolado)  
- Testes de integração: avaliam o ecossistema em que os métodos se conectam, executados localmente, em pipeline ou a cada commit  
- Testes de UI: observam alinhamento, tipografia, iconografia, ancoragem e vocabulário da interface, com cooperação entre qualidade, dev e design  
- Garantia de qualidade: cobertura de requisitos funcionais e não funcionais, processos de build (commit/validação/teste completo), review e merge com camada de qualidade  
- Métricas de qualidade: impacto na qualidade e no tempo de desenvolvimento, independência entre testes e feedback rápido para evitar bugs  
- Boas práticas: reconhecer a unidade como a parte indivisível do código, manter consistência visual na experiência do usuário e integrar ecossistema front-end/back-end  

# Ferramentas iniciais

Ferramentas para testes unitários/integração/UI (ex: stubs e esqueletos para isolamento), pipelines de integração contínua e frameworks como JUnit/Jest para TDD/BDD
